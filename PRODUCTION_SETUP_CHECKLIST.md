## 4. Supabase Mobile and Web Auth Setup

### Values
| Item | Where Used | Public? | Where to Find |
|---|---|---|---|
| Project URL (`SUPABASE_URL`) | Server + frontend | Yes | Supabase dashboard → Project Settings → API |
| Anon Key (`SUPABASE_ANON_KEY`) | Server + frontend | Yes (by design) | Supabase dashboard → Project Settings → API |
| Service Role Key (`SUPABASE_SERVICE_ROLE_KEY`) | Server only | ✅ SECRET | Supabase dashboard → Project Settings → API |

### Auth Redirect URLs
Add all of the following to **Supabase → Authentication → URL Configuration → Redirect URLs**:

```
https://www.bid-reel.com/**
https://bid-reel.com/**
https://bid-reel.vercel.app/**
http://localhost:5173/**
com.bidreel.app://callback          ← Android deep link (if using OAuth)
```

### Mobile (Capacitor Android)
- Auth uses phone OTP via Supabase — no OAuth redirect needed for the current login flow
- `allowNavigation: ["*.supabase.co"]` is already set in `capacitor.config.ts`
- If OAuth (Google, Apple) is added later, update the redirect URLs above and add the custom scheme intent filter to `AndroidManifest.xml`

### Security Notes
- `SUPABASE_SERVICE_ROLE_KEY` bypasses Row Level Security — **never** put it in the frontend bundle or `VITE_*` vars
- The anon key is safe to embed in the frontend (it is subject to RLS)

## 5. Cloudflare R2 Secure Media Configuration

### Upload Flow (Current Architecture)
```
Browser → POST /api/media/upload-url (Vercel serverless, authenticated)
        ← Returns presigned PUT URL (signed with R2 credentials server-side)
Browser → PUT {presigned URL} directly to R2 (no secrets in browser)
```
This is the **correct and safe pattern** — R2 credentials never reach the browser.

### Required Config
| Item | Variable | Secret? | Where to Set |
|---|---|---|---|
| Account ID | `R2_ACCOUNT_ID` | No | Vercel env vars |
| Bucket name | `R2_BUCKET` | No | Vercel env vars |
| S3 endpoint | Derived: `https://{R2_ACCOUNT_ID}.r2.cloudflarestorage.com` | No | Auto-built in code |
| Public CDN URL | `R2_PUBLIC_BASE_URL` | No | Vercel env vars |
| Access key | `R2_ACCESS_KEY_ID` | ✅ SECRET | Vercel env vars |
| Secret key | `R2_SECRET_ACCESS_KEY` | ✅ SECRET | Vercel env vars |

### R2 Dashboard Steps
1. **API Token**: Cloudflare → R2 → Manage API Tokens → Create token with **Object Read & Write** on `bidreel-media` bucket
2. **CORS Policy** on `bidreel-media` bucket:
```json
[{
  "AllowedOrigins": ["https://bid-reel.com", "https://www.bid-reel.com", "https://bid-reel.vercel.app", "http://localhost:5173"],
  "AllowedMethods": ["PUT"],
  "AllowedHeaders": ["*"],
  "MaxAgeSeconds": 86400
}]
```
3. **Public access**: Bucket must have public development URL enabled, OR a custom domain set on the bucket (update `R2_PUBLIC_BASE_URL` accordingly)

### Known Issue to Watch
When pasting `R2_ACCESS_KEY_ID` into Vercel: confirm the value is exactly **32 characters** with no trailing newline. A 33-character key causes `InvalidArgument` (R2 rejects the presigned URL silently, browser sees a network error).
