أنت مهندس برمجيات خبير في Vite وVercel وmonorepo.

أريدك أن تشخّص وتصلّح مشكلة الـ build في هذا المشروع بشكل كامل، مع أقل تغييرات ممكنة، وبشكل متوافق مع:

* GitHub
* Vercel
* Root Directory الحالي على Vercel: `artifacts/bidreel-web`
* المشروع يجب أن يعمل كويب الآن، ولا يكسر قابلية تحويله لاحقًا إلى Android عبر Capacitor

المشكلة الحالية على Vercel هي:

`failed to load config from /vercel/path0/artifacts/bidreel-web/vite.config.ts`
`Error: BASE_PATH environment variable is required but was not provided.`

ومعطيات مهمة:

* المشروع monorepo
* الواجهة موجودة في: `artifacts/bidreel-web`
* Vercel مربوط على هذا المسار
* يوجد `vercel.json` في جذر المستودع
* سابقًا كان عندنا أيضًا مشاكل مرتبطة بـ outputDirectory
* المطلوب إصلاح حقيقي داخل الكود/الإعدادات، وليس فقط اقتراحات نظرية

المطلوب منك بالترتيب:

1. افحص الملفات التالية بدقة:

* `artifacts/bidreel-web/vite.config.ts`
* `artifacts/bidreel-web/package.json`
* `vercel.json`
* أي ملفات env أو config مرتبطة بالويب

2. حدّد لماذا `BASE_PATH` مطلوب إجباريًا، وهل هذا منطقي في بيئة Vercel أم لا.

3. أصلح المشكلة بأفضل طريقة احترافية:

* إذا كان الأفضل جعل `BASE_PATH` اختياريًا، عدّل الكود ليستخدم `/` كقيمة افتراضية آمنة
* إذا كان هناك متغيرات أخرى قد تسبب فشلًا مشابهًا على Vercel، عالجها أيضًا
* لا تكسر المسارات الداخلية ولا الـ routing ولا الأصول الثابتة

4. تحقّق أيضًا من توافق الإعدادات مع:

* Vercel root directory = `artifacts/bidreel-web`
* output directory الصحيح
* build command الصحيح

5. إذا وجدت بقايا إعدادات خاصة بـ Replit أو بيئة قديمة تعطل Vercel، نظّفها أو اجعلها متوافقة.

6. نفّذ التعديلات المطلوبة مباشرة على الملفات.

7. بعد الإصلاح، أعطني تقريرًا مختصرًا جدًا يتضمن:

* ما سبب المشكلة
* ما الملفات التي عدلتها
* ما القيم الافتراضية الجديدة
* هل أصبح المشروع جاهزًا للـ deploy على Vercel

مهم جدًا:

* لا تغيّر البنية العامة للمشروع بدون حاجة
* لا تحذف ميزات Android/Capacitor
* لا تقدّم شرحًا طويلًا قبل التنفيذ
* ابدأ أولًا بفحص `vite.config.ts` ثم أصلح المشكلة من المصدر
