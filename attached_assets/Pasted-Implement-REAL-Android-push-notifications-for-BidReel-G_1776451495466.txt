Target:

* Capacitor Android app

## PART 1 — ANDROID / CAPACITOR SETUP

Set up real push support for the Android app:

1. install and configure the Capacitor push notifications plugin
2. wire Firebase Cloud Messaging correctly
3. ensure Android project includes the required Firebase config
4. request push permission on app startup or first eligible session
5. register the device and obtain the FCM token
6. handle token refresh
7. handle foreground notification reception
8. handle notification tap to deep-link into the correct screen

## PART 2 — DEVICE TOKEN STORAGE

Build server-side token persistence.

Requirements:

1. create a table for push tokens if not already present
2. store:

   * user_id
   * push token
   * platform
   * created_at
   * updated_at
   * is_active
3. add an authenticated endpoint for registering/updating a token
4. deactivate old tokens when needed
5. support multiple devices per user safely

Do not store push tokens only in local storage. They must be persisted server-side.

---

## PART 3 — SERVER PUSH SENDER

Build a real push sending service.

Requirements:

1. create a server-side push notification sender using FCM
2. send push only for high-value notification types
3. include:

   * title
   * body
   * data payload
   * deep link target
4. do not break in-app notifications
5. push must be additive to existing in-app notifications, not a replacement

---

## PART 4 — EVENT WIRING

Wire push sending into the real backend event flows.

At minimum:

* when seller receives a new bid → send bid_received push
* when previous highest bidder is outbid → send outbid push
* when winner is determined → send auction_won push
* when someone follows a user → send followed_you push

Each event must:

1. create the in-app notification row
2. then send push if the user has active tokens

No self-notifications.
No duplicate spam.
