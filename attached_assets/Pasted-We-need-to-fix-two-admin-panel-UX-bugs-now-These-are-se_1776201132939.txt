* in the admin users page and admin auctions page
* when clicking the three-dots action button
* the menu opens in a broken/clipped way
* part of the screen becomes awkwardly covered
* instead of a normal dropdown opening downward and staying within the viewport

### Required audit

Inspect:

1. the exact menu/dropdown component used in users and auctions pages
2. whether it is rendered inside an overflow-hidden container, table wrapper, or clipped parent
3. whether z-index / fixed / absolute positioning is wrong
4. whether RTL positioning is contributing to the bug
5. whether the menu should use a portal/popover strategy instead of rendering inside the table

### Required fix

Make the action menu open cleanly and predictably in both:

* admin users page
* admin auctions page

Expected behavior:

* clicking the three-dots button opens a compact action menu
* the menu stays fully visible
* it should open downward or in the safest direction depending on viewport space
* it must not clip into the table container
* it must not cover half the page
* it must behave correctly in RTL

If a shared dropdown/menu component is used, fix it at the shared level if safe.
